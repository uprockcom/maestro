# Container File Reference

This document catalogs every special file inside a Maestro container — what creates it, when, and why. Files are grouped by lifecycle phase.

## Directory Layout

```
/home/node/
├── .claude/
│   ├── CLAUDE.md                    # Agent-facing docs (auto-discovered by Claude Code)
│   ├── settings.json                # Claude Code hooks configuration
│   ├── .credentials.json            # OAuth token (copied from host)
│   ├── debug/                       # Container-specific logs (auto-generated)
│   └── statsig/                     # Container-specific telemetry (auto-generated)
├── .claude.json                     # Claude Code config (copied from host, note: in home, not .claude/)
├── .maestro/
│   ├── MAESTRO.md                   # Agent-facing docs (canonical location)
│   ├── claude-idle                  # Flag file: Claude is waiting for input
│   ├── current-question.json        # Structured AskUserQuestion data (when active)
│   ├── daemon/
│   │   └── daemon-ipc.json          # Daemon connection info (read-only mount from host)
│   └── requests/
│       └── {uuid}.json              # IPC request files
├── .config/gh/                      # GitHub CLI config (optional, copied from host)
├── .gitconfig                       # Git identity (maestro@container)
├── .tmux.conf                       # Tmux true-color and status line config
├── .zshrc                           # Shell config (powerline10k + custom prompt)
└── go/                              # GOPATH

/workspace/                          # Project files + .git (copied from host cwd)
/workspace/../<sibling>/             # Additional folders (from config, optional)

/etc/
├── allowed-domains.txt              # Firewall domain whitelist
├── internal-dns.txt                 # Internal DNS server IP (optional)
├── internal-domains.txt             # Internal domain list (optional)
├── aws-enabled.txt                  # AWS access flag (optional)
├── tmux.conf                        # Global tmux config (baked in image)
├── sudoers.d/node                   # Sudo permissions for node user
└── resolv.conf                      # Rewritten to 127.0.0.1 by firewall

/tmp/
├── dnsmasq-firewall.conf            # dnsmasq config (generated by firewall, updated by add-domain)
├── dnsmasq.log                      # DNS query log
├── auto-input.sh                    # One-shot script to inject initial prompt
├── prompt-input.txt                 # Initial task prompt (consumed by auto-input.sh)
└── maestro-msg                      # Transient: message being sent to Claude via tmux

/usr/local/bin/
├── daemon-ipc                       # Privilege-isolated IPC proxy binary
├── maestro-request                  # Container-side IPC client CLI
├── init-firewall.sh                 # Firewall/DNS setup script
└── container-startup.sh             # Container init script (CMD entrypoint)
```

---

## Phase 1: Baked into Docker Image

These files exist in every container from the moment it's created.

### Binaries

| Path | Source | Purpose |
|---|---|---|
| `/usr/local/bin/daemon-ipc` | `docker/daemon-ipc/main.go` (CGO_ENABLED=0 go build) | Privilege-isolated IPC proxy. Runs as `maestro-ipc` user. Only process allowed through firewall to reach host. |
| `/usr/local/bin/maestro-request` | `docker/maestro-request-go/` (CGO_ENABLED=0 go build) | Container-side CLI for IPC: `new`, `done`, `notify`, `status`, `wait`, `claude` subcommands. |
| `/usr/local/bin/init-firewall.sh` | `assets/init-firewall.sh` | Firewall setup: iptables rules, ipset whitelist, dnsmasq DNS filtering, GitHub IP detection. |
| `/usr/local/bin/container-startup.sh` | `docker/container-startup.sh` | CMD entrypoint. Fixes home dir ownership, installs custom CAs, updates Claude Code, sets prompt. |

### System Configuration

| Path | Created By | Purpose |
|---|---|---|
| `/etc/tmux.conf` | Dockerfile `RUN echo` | Mouse support, 50k history, status line settings, window-swap keybindings. |
| `/etc/sudoers.d/node` | Dockerfile `RUN echo` (perms `0440`) | Passwordless sudo for: init-firewall.sh, dnsmasq, tee resolv.conf, chown, chmod, update-ca-certificates, daemon-ipc (as maestro-ipc). |
| `/etc/apt/keyrings/githubcli-archive-keyring.gpg` | Dockerfile `RUN wget` | GPG key for GitHub CLI apt repository. |
| `maestro-ipc` system user | Dockerfile `RUN useradd` | Dedicated user for daemon-ipc. Enables iptables uid-owner matching for firewall isolation. |

### Pre-created Directories

| Path | Owner | Purpose |
|---|---|---|
| `/workspace/` | node:node | Project workspace (populated at startup). |
| `/home/node/.claude/` | node:node | Claude state directory. |
| `/home/node/go/` | node:node | GOPATH. |
| `/usr/local/share/npm-global/` | node:node | Global npm packages. |
| `/commandhistory/` | node | Persistent bash/zsh history. |

### Installed Runtimes

| Runtime | Version | Path |
|---|---|---|
| Node.js | 20.x | Base image |
| Go | 1.26.0 | `/usr/local/go/` |
| Java JDK | 17.0.2 | `/usr/local/jdk-17.0.2/` |
| Gradle | 8.13 | `/usr/local/gradle-8.13/` |
| Python 3 | System | `/usr/bin/python3` |
| UV | Latest | `/home/node/.cargo/bin/uv` |
| Claude Code | Configurable | `/usr/local/share/npm-global/bin/claude` |
| gh CLI | Latest (GitHub repo) | `/usr/bin/gh` |

## Phase 2: Created at Container Startup

These files are created by `cmd/new.go` functions after the container starts but before Claude begins.

### Authentication & Config (copied from host)

| Path | Source on Host | Function | Notes |
|---|---|---|---|
| `/home/node/.claude/.credentials.json` | `~/.maestro/.claude/.credentials.json` | `startContainer()` | OAuth token. Ownership fixed to node:node. |
| `/home/node/.claude.json` | `~/.maestro/.claude/.claude.json` | `startContainer()` | Claude Code config. Note: in home dir, NOT `.claude/`. |
| `/home/node/.config/gh/` | `~/.config/gh/` (configurable) | `startContainer()` | GitHub CLI auth. Only if `github.enabled: true` in config. |

### Agent Documentation

| Path | Function | Format | Notes |
|---|---|---|---|
| `/home/node/.maestro/MAESTRO.md` | `writeMaestroMD()` | Markdown | Template variables replaced: `{{CONTAINER_NAME}}`, `{{BRANCH_NAME}}`, `{{MAESTRO_VERSION}}`, `{{PARENT_CONTAINER}}`. |
| `/home/node/.claude/CLAUDE.md` | `writeMaestroMD()` | Markdown | Same content as MAESTRO.md. Placed here for Claude Code auto-discovery. |

### Claude Code Hooks

| Path | Function | Format |
|---|---|---|
| `/home/node/.claude/settings.json` | `writeClaudeSettings()` | JSON |

Hook events configured:

| Event | Matcher | Action |
|---|---|---|
| `Stop` | (all) | `touch claude-idle` |
| `UserPromptSubmit` | (all) | `rm -f claude-idle` |
| `PreToolUse` | (all) | `rm -f claude-idle` |
| `PreToolUse` | `AskUserQuestion\|EnterPlanMode\|ExitPlanMode` | `touch claude-idle` |
| `PreToolUse` | `AskUserQuestion` | `jq '.tool_input' > current-question.json` |
| `PostToolUse` | `AskUserQuestion\|EnterPlanMode\|ExitPlanMode` | `rm -f claude-idle` |
| `PostToolUse` | `AskUserQuestion` | `rm -f current-question.json` |
| `PostToolUseFailure` | `AskUserQuestion` | `rm -f current-question.json` |

### Tmux & Session Setup

| Path | Function | Purpose |
|---|---|---|
| `/home/node/.tmux.conf` | `startTmuxSession()` | True-color support, status bar with container name + branch + time. |
| `/tmp/auto-input.sh` | `startTmuxSession()` | One-shot background script: waits for Claude startup, accepts permissions, sends initial prompt via tmux paste-buffer. |
| `/tmp/prompt-input.txt` | Created by auto-input.sh | The task planning prompt text. Consumed immediately. |

### Shell Configuration

| Path | Function | Purpose |
|---|---|---|
| `/home/node/.zshrc` | `startContainer()` (modified via sed + append) | Disables powerline10k, adds custom git-aware prompt with colors and status indicators. |

### Directories

| Path | Function | Purpose |
|---|---|---|
| `/home/node/.maestro/` | `startContainer()` | IPC state root. |
| `/home/node/.maestro/requests/` | `startContainer()` | Request file storage. |

### Firewall Configuration

| Path | Function | Format | Notes |
|---|---|---|---|
| `/etc/allowed-domains.txt` | `initializeFirewall()` | Text (one domain/line) | From `config.Firewall.AllowedDomains`. |
| `/etc/internal-dns.txt` | `initializeFirewall()` | Text (single IP) | Optional. For corporate DNS (Zscaler, VPN). |
| `/etc/internal-domains.txt` | `initializeFirewall()` | Text (one domain/line) | Optional. Domains routed via internal DNS. |
| `/etc/aws-enabled.txt` | `initializeFirewall()` | Text ("enabled") | Optional. Signals firewall to allow AWS domains. |
| `/tmp/dnsmasq-firewall.conf` | `init-firewall.sh` | dnsmasq config | Generated from allowed-domains.txt. ipset rules + upstream DNS. Updated by `maestro add-domain`. |
| `/tmp/dnsmasq.log` | dnsmasq process | Log file | DNS query log for debugging. |
| `/etc/resolv.conf` | `init-firewall.sh` | Text | Rewritten to `nameserver 127.0.0.1` (local dnsmasq). |

### Project Files

| Path | Function | Notes |
|---|---|---|
| `/workspace/*` | `copyProjectToContainer()` | Tar-piped from host cwd. Excludes node_modules, .git (copied separately). |
| `/workspace/.git/` | `copyProjectToContainer()` | Copied via `docker cp` after main tar. |
| `/workspace/../<sibling>/` | `copyAdditionalFolders()` | From `config.Sync.AdditionalFolders`. Optional. |

## Phase 3: Runtime Files

Created during Claude's session by hooks, maestro-request commands, or the daemon.

### Idle Detection Flag

| Path | Format | Created By | Deleted By |
|---|---|---|---|
| `/home/node/.maestro/claude-idle` | Empty flag file | `Stop` hook, `PreToolUse` hook (blocking tools) | `UserPromptSubmit` hook, `PreToolUse` hook (general), `PostToolUse` hook (blocking tools), daemon `send_message` handler |

The daemon checks this flag + Claude process status to determine container attention state:
- Flag exists + Claude running = **needs attention** (idle, waiting for input)
- Flag exists + Claude not running = **stale flag** (dormant, ignore)
- Flag absent = **active** (working)

### Question Data Capture

| Path | Format | Created By | Deleted By |
|---|---|---|---|
| `/home/node/.maestro/current-question.json` | JSON | `PreToolUse` hook (AskUserQuestion) | `PostToolUse` hook (AskUserQuestion), `PostToolUseFailure` hook (AskUserQuestion) |

Contains the `tool_input` from AskUserQuestion — the structured question data with questions array, options, headers, and multiSelect flags. Allows external tools to display questions without parsing tmux output.

### IPC Request Files

| Path | Format | Created By | Updated By |
|---|---|---|---|
| `/home/node/.maestro/requests/{uuid}.json` | JSON | `maestro-request` commands | Daemon (via `docker exec tee`) |

**Request file structure:**

```json
{
  "id":                "uuid-v4",
  "action":            "new | notify | exit | wait_idle | read_messages | send_message",
  "status":            "pending | fulfilled | failed | child_exited",
  "parent":            "container-hostname",
  "requested_at":      "2025-01-01T00:00:00Z",
  "fulfilled_at":      "2025-01-01T00:00:05Z | null",
  "error":             "error message | null",
  "task":              "task description (new only)",
  "branch":            "branch name (new only, optional)",
  "child_container":   "container-name (new only, set on fulfill)",
  "child_exited_at":   "2025-01-01T00:01:00Z (new only, set on exit)",
  "title":             "notification title (notify only)",
  "message":           "message body (notify/send_message)",
  "target_request_id": "uuid (wait_idle/read_messages/send_message)",
  "messages":          [{"role":"...", "content":"...", "timestamp":"..."}],
  "count":             10,
  "timeout":           300
}
```

**Status lifecycle:** `pending` → `fulfilled` → `child_exited` (for `new` action), or `pending` → `failed`.

### Daemon IPC Config (mounted read-only)

| Path | Format | Created By | Notes |
|---|---|---|---|
| `/home/node/.maestro/daemon/daemon-ipc.json` | JSON | Daemon on host | Read-only bind mount. Contains `port`, `bridge_port`, `token`, `pid`. Used by `daemon-ipc` binary to reach the host daemon. |

### Transient Files

| Path | Created By | Lifetime | Purpose |
|---|---|---|---|
| `/tmp/maestro-msg` | Daemon `send_message` handler | Seconds | Message text loaded into tmux paste-buffer, then deleted. |
| `/tmp/host-certs/*` | `copySSLCertificates()` | Startup only | Temp copy of host SSL certs. Deleted after Java keystore import. |
